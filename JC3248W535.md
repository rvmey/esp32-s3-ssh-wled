# Guition JC3248W535 — 320×480 Screen via SSH

This variant targets the **Guition JC3248W535** (ESP32-S3, 320×480 AXS15231
QSPI display) and fills the entire screen with a solid colour in response to
SSH commands.

---

## Shell commands

```
> help

Commands:
  color <name>      Named colour: red green blue white yellow
                    cyan magenta purple orange pink
  color R G B       RGB triplet, each value 0-255
  color #RRGGBB     Hex colour (e.g. #FF8800)
  off               Fill screen black
  status            Show current screen colour
  help              Show this help text
  exit | quit       Close the connection
```

### Examples

```
> color red
Screen set to R:255 G:0   B:0    (#FF0000)

> color 0 128 255
Screen set to R:0   G:128 B:255  (#0080FF)

> color #FF8800
Screen set to R:255 G:136 B:0    (#FF8800)

> off
Screen off.

> status
Screen is off (black).
```

## Boot indicators

The screen itself is used as the boot status indicator:

| Colour | Meaning |
|--------|---------|
| Brief blue fill | Wi-Fi provisioning or connecting |
| Brief green fill | Wi-Fi connected, IP obtained |
| Solid red fill | Error (Wi-Fi failed or SSH init failed) — check serial log |

| Backlight | 1 | Driven high at init (full brightness) |
| Display controller | AXS15231 | 320×480, RGB565, 40 MHz QSPI |

---

## Display driver notes

- Pixel format: **RGB565** (16-bit, big-endian), `COLMOD = 0x55`
- Screen is filled one row at a time using RAMWR (`0x2C`) for the first row
  and RAMWRC (`0x3C`) for subsequent rows, both with the quad-wire prefix
  `0x32` so pixel data transfers at 4× the clock rate.
- The AXS15231 vendor init sequence (unlock / config / lock via register
  `0xBB`) is sent once on boot; see `screen_control.c` for details.
- Backlight is configured as a GPIO output (full on). PWM dimming can be
  added later using the LEDC peripheral on GPIO 1.

## Building this variant

In `menuconfig`, under **"ESP32 SSH LED Configuration"**, select:

> **Hardware variant → Guition JC3248W535 (320×480 AXS15231 QSPI display)**

Then build normally:

```bash
idf.py build
idf.py -p COM<N> flash
```

## Troubleshooting

| Symptom | Likely cause |
|---------|-------------|
| Screen stays black | Backlight GPIO 1 not driven — check power rail; confirm `screen_init()` completed |
| Screen shows noise / wrong colours | SPI clock too high — try reducing `LCD_CLK_HZ` to 20 MHz in `screen_control.c` |
| Display draws slowly | Enable `CONFIG_SPI_MASTER_IN_IRAM=y` in `sdkconfig.defaults` |
| Garbled image | `draw_rounding` misalignment — the AXS15231 requires column/row addresses aligned to 8 pixels |
